Example 1

This example has no support for dark mode. When dark mode is enabled via browser or operating system settings, the example will not change. Dark mode is an enhancement and opt-in only, as it should be. If we do not include support for it, site will remain unchanged. Always.

Example 2

This example does support dark mode. When dark mode is enabled via browser or operating system settings, the example will change to a black background with white text. The image will also contain a white border. Here, we are choosing to support dark mode and utilizing prefers-color-scheme: dark to achieve this effect.

Note: To enable dark mode on Windows 11 go to Start > Settings > Personalization > Colors > Choose your default app mode (Dark).

Browser Extensions

Browser extension do not enable true dark mode. Many simply override the default stylesheet with their own sets of styles, often with the !important declaration included. Radancy should not be attempting to overrride third-party extension that may alter the page. Only true dark mode will be honored, if needed. However, this may come at an additional cost.
