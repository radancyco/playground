Tracking Data Interfering With preventDefault() Method

Issue: Something in the tracking code appears to be interfering with the preventDefault() method applied to hyperlinks. This should be looked into further as supressing default behavior of hyperlinks is a very common practice for the delivery team to implement. In the following examples, I have preventDefault() applied to both links via an onclick event. Again, very common. The first link includes the tracking data attributes, while the second does not. When cliking on the first link, we are seeing the URL fragment appended to the URL, which should not be occuring. The preventDefault() method is being bypassed in some way. In the second example, the link is properly supressed and no fragment is appended to the URL.
